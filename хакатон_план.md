* Docker, если нет обязательного backend service.
* Three.js, если тема не требует 3D.
* Multiplayer real-time.
* Сложную регистрацию.
* Redux.
* Prisma migrations ради одной таблицы.
* Godot/Unity WebGL, если deploy должен быть быстрым.
* Собственный sprite editor.
* Agent swarm без главного интегратора.

## 4. AI-интеграция через DeepSeek

DeepSeek API совместим с OpenAI-форматом, официальный `base_url` для OpenAI-style вызовов: `https://api.deepseek.com`. В документации сейчас указаны модели `deepseek-v4-flash`, `deepseek-v4-pro`, а также совместимые старые имена `deepseek-chat` и `deepseek-reasoner`; важно, что `deepseek-chat` и `deepseek-reasoner` отмечены как deprecated с 24 июля 2026, поэтому держи модель в env. ([DeepSeek API Docs][2])

### Как получить API key

1. Зарегистрируйся в DeepSeek Platform.
2. Создай API key.
3. Сохрани ключ только в `.env.local`.
4. Добавь тот же ключ в Vercel Environment Variables. Vercel хранит env values вне кода и применяет их к build/runtime окружениям. ([Vercel][4])

### `.env.local`

```env
DEEPSEEK_API_KEY=
DEEPSEEK_BASE_URL=https://api.deepseek.com
AI_MODEL=deepseek-chat
AI_PROVIDER=deepseek

OPENROUTER_API_KEY=
OPENAI_API_KEY=
GEMINI_API_KEY=

AI_MAX_TOKENS=220
AI_TIMEOUT_MS=8000
AI_DAILY_BUDGET_REQUESTS=120
```

### Почему нельзя `NEXT_PUBLIC_DEEPSEEK_API_KEY`

В Next.js переменные без `NEXT_PUBLIC_` доступны только на сервере; переменные с `NEXT_PUBLIC_` встраиваются в client bundle и становятся видимыми браузеру. Поэтому API key никогда не должен иметь `NEXT_PUBLIC_`. ([nextjs.org][5])

### Server-side route: `app/api/ai/route.ts`

```ts
import { NextRequest, NextResponse } from "next/server";

export const runtime = "nodejs";

type AiMode =
  | "quest"
  | "npc"
  | "coach"
  | "battle"
  | "ending"
  | "item"
  | "judge";

const fallbackByMode: Record<AiMode, string> = {
  quest: "A strange signal appears nearby. Choose: investigate, hide, or trade resources.",
  npc: "I do not trust you yet, traveler. Prove your intent with one smart move.",
  coach: "You played aggressively. Next run, preserve resources during the first half.",
  battle: "The clash is brief, loud, and decisive. Your timing changes the outcome.",
  ending: "Your run becomes a local legend: imperfect, risky, but unforgettable.",
  item: "Echo Compass: reveals one hidden event, but costs 1 energy.",
  judge: "Score: 78. Strong strategy, but weak resource control."
};

function safeMode(value: unknown): AiMode {
  const allowed: AiMode[] = ["quest", "npc", "coach", "battle", "ending", "item", "judge"];
  return allowed.includes(value as AiMode) ? (value as AiMode) : "coach";
}

function buildSystemPrompt(mode: AiMode) {
  return `
You are the AI gameplay engine for a 5-minute web game prototype.
Return concise, vivid, gameplay-useful text.
No markdown. No long explanations.
Mode: ${mode}.
Max 3 short sentences.
`;
}

async function callOpenAICompatible(params: {
  baseUrl: string;
  apiKey: string;
  model: string;
  mode: AiMode;
  prompt: string;
  timeoutMs: number;
  maxTokens: number;
}) {
  const controller = new AbortController();
  const timeout = setTimeout(() => controller.abort(), params.timeoutMs);

  try {
    const res = await fetch(`${params.baseUrl.replace(/\/$/, "")}/v1/chat/completions`, {
      method: "POST",
      signal: controller.signal,
      headers: {
        "Content-Type": "application/json",
        Authorization: `Bearer ${params.apiKey}`
      },
      body: JSON.stringify({
        model: params.model,
        messages: [
          { role: "system", content: buildSystemPrompt(params.mode) },
          { role: "user", content: params.prompt.slice(0, 1200) }
        ],
        temperature: 0.8,
        max_tokens: params.maxTokens,
        stream: false
      })
    });

    if (!res.ok) {
      throw new Error(`AI provider failed: ${res.status}`);
    }

    const data = await res.json();
    return data?.choices?.[0]?.message?.content?.trim();
  } finally {
    clearTimeout(timeout);
  }
}

export async function POST(req: NextRequest) {
  try {
    const body = await req.json();
    const mode = safeMode(body.mode);
    const prompt = String(body.prompt ?? "").slice(0, 1200);

    if (!prompt) {
      return NextResponse.json({ text: fallbackByMode[mode], fallback: true });
    }

    const timeoutMs = Number(process.env.AI_TIMEOUT_MS ?? 8000);
    const maxTokens = Math.min(Number(process.env.AI_MAX_TOKENS ?? 220), 300);

    const providers = [
      {
        name: "deepseek",
        baseUrl: process.env.DEEPSEEK_BASE_URL ?? "https://api.deepseek.com",
        apiKey: process.env.DEEPSEEK_API_KEY,
        model: process.env.AI_MODEL ?? "deepseek-chat"
      },
      {
        name: "openrouter",
        baseUrl: "https://openrouter.ai/api/v1",
        apiKey: process.env.OPENROUTER_API_KEY,
        model: "deepseek/deepseek-chat"
      },
      {
        name: "openai",
        baseUrl: "https://api.openai.com",
        apiKey: process.env.OPENAI_API_KEY,
        model: "gpt-4o-mini"
      }
    ];

    for (const provider of providers) {
      if (!provider.apiKey) continue;

      try {
        const text = await callOpenAICompatible({
          baseUrl: provider.baseUrl,
          apiKey: provider.apiKey,
          model: provider.model,
          mode,
          prompt,
          timeoutMs,
          maxTokens
        });

        if (text) {
          return NextResponse.json({
            text,
            provider: provider.name,
            fallback: false
          });
        }
      } catch {
        // Try next provider.
      }
    }

    return NextResponse.json({
      text: fallbackByMode[mode],
      fallback: true
    });
  } catch {
    return NextResponse.json(
      { text: "The AI signal is unstable. Continue with the fallback event.", fallback: true },
      { status: 200 }
    );
  }
}
```

### Как ограничить расходы

* `max_tokens: 150-300`.
* `prompt.slice(0, 1200)`.
* Не вызывать AI каждый кадр, только на события: старт квеста, NPC, конец раунда.
* Кэшировать по ключу: `mode + gameStateHash`.
* Лимит на сессию: 5-8 AI calls.
* Local fallback всегда включен.
* В UI показывать маленький бейдж: `AI Live` или `Fallback Mode`.
* DeepSeek pricing держать открытым перед стартом, потому что цены и модели могут обновляться; в официальной таблице сейчас есть отдельные цены для cache hit, cache miss и output tokens. ([DeepSeek API Docs][6])

### Как сделать AI видимым для жюри

Плохо: “У нас где-то AI генерирует текст”.
Хорошо: “Каждый раунд AI анализирует стиль игрока и меняет следующий challenge”.

UI элементы:

* “AI Director is watching your choices”.
* “Generated quest”.
* “Personalized coach”.
* “Fallback mode active” если API упал.
* Финальный AI recap: 3 предложения, оценка, совет.

## 6. Если тема будет RPG

### Blueprint: “AI Micro-RPG Run”

Core loop:

1. Игрок выбирает героя.
2. Получает 3 stats: `Health`, `Energy`, `Trust`.
3. Каждый ход: событие, 3 выбора.
4. Выбор меняет stats.
5. AI Narrator адаптирует следующее событие.
6. После 8-10 ходов: boss/event.
7. Концовка + score + leaderboard.

### Player stats

```ts
type PlayerStats = {
  hp: number;
  energy: number;
  gold: number;
  trust: number;
  danger: number;
  score: number;
};
```

### Enemies

* `Scout`: low hp, teaches combat.
* `Trickster`: steals resource.
* `Guardian`: requires item/trust.
* `Final Threat`: generated by AI based on choices.

### Map/events

Не делай настоящую карту. Делай **node map illusion**:

```txt
Camp -> Ruins -> Market -> Rift -> Boss
```

Каждый node: card, background, choices.

### Inventory

5 типов предметов:

* heal;
* reveal;
* shield;
* double score;
* reroll event.

### Win/lose

Win: пройти 8 ходов и пережить финальное событие.
Lose: hp <= 0 или danger >= 10.
Partial win: особая концовка за высокий trust/score.

### Leaderboard

Сначала local:

```ts
[{ name: "YOU", score: 1240, ending: "The Signal Keeper" }]
```

Потом Supabase if time.

### Daily challenge

Seed от даты:

```ts
const dailySeed = new Date().toISOString().slice(0, 10);
```

### README pitch

> A 5-minute AI-directed RPG where every choice changes the narrator, difficulty, and ending. Built for fast replayable story runs with leaderboard-driven daily challenges.

## 10. Правила разработки, чтобы не потерять время

### Жесткий protocol

* Review: максимум 5 минут на блок.
* Рефакторинг: только если мешает shipping.
* Архитектуру нельзя менять после 01:30.
* Commit каждые 45 минут.
* Deploy первый раз до 00:30.
* Final deploy до 04:30.
* Последние 30 минут: только баги, README, pitch.

### Если тесты падают

1. Если build падает, чинить.
2. Если lint падает, временно ослабить rule.
3. Если type issue не влияет на runtime, быстро типизировать `unknown`/safe fallback.
4. Если модуль сломан, отключить фичу, не чинить 40 минут.

### Если AI API не работает

* Включить fallback mode.
* В UI показать `AI fallback simulation`.
* В pitch сказать: “AI route has provider fallback and deterministic offline mode”.
* Не тратить больше 15 минут подряд.

### Если backend не успеваешь

* LocalStorage leaderboard.
* Fake profile name.
* Export result as share text.
* Server route оставить только для AI.

### Если дизайн слабый

Добавить:

* темный фон;
* 2xl cards;
* glow/border;
* Lucide icons;
* Motion появление карточек;
* один сильный цвет;
* result modal.

## 11. Backend/database strategy

### Когда использовать Supabase

Используй Supabase, если:

* нужен реальный online leaderboard;
* нужно хранить sessions;
* есть 30-40 свободных минут после core loop;
* у команды есть человек, который уже умеет Supabase.

Supabase имеет Next.js quickstart и auth flow, но для хакатона лучше брать только database/leaderboard, а auth оставить на потом. ([Supabase][7])

### Когда не использовать базу

Не использовать DB, если:

* игра еще не playable;
* осталось меньше 90 минут;
* leaderboard можно показать local;
* тема не требует аккаунтов.

### Минимальная схема

```sql
create table profiles (
  id uuid primary key default gen_random_uuid(),
  display_name text not null,
  created_at timestamp with time zone default now()
);

create table leaderboard (
  id uuid primary key default gen_random_uuid(),
  display_name text not null,
  score int not null,
  ending text,
  created_at timestamp with time zone default now()
);

create table game_sessions (
  id uuid primary key default gen_random_uuid(),
  display_name text,
  score int not null,
  result jsonb not null,
  ai_used boolean default false,
  created_at timestamp with time zone default now()
);
```

### Local-only fallback

```ts
export function saveLocalScore(entry: ScoreEntry) {
  const key = "hackathon-game-leaderboard";
  const current = JSON.parse(localStorage.getItem(key) ?? "[]");
  const next = [...current, entry]
    .sort((a, b) => b.score - a.score)
    .slice(0, 10);
  localStorage.setItem(key, JSON.stringify(next));
  return next;
}
```

### Что можно имитировать для демо

Можно:

* local leaderboard;
* demo profile;
* seed daily challenge;
* sample analytics panel.

Нельзя фейкать:

* live API, если говоришь “real AI”.
* online leaderboard, если жюри проверит с другого устройства.
* auth, если кнопка “sign in” никуда не ведет.

## 14. Полезные актуальные инструменты и ссылки

### Must-have

* **Next.js App Router**: server routes, pages, deploy-friendly architecture. Route Handlers живут в `app` и поддерживают HTTP methods. ([nextjs.org][1])
* **Vercel**: Git integration дает preview deployments и production deployments через main branch. ([Vercel][3])
* **DeepSeek API**: OpenAI-compatible, DeepSeek-first AI route. ([DeepSeek API Docs][2])
* **Tailwind CSS**: быстрый styling в Next.js. ([tailwindcss.com][8])
* **shadcn/ui**: copyable components, open code, хорошо подходит для AI-assisted UI. ([ui.shadcn.com][9])
* **Zustand**: маленький state management без лишнего ceremony. ([zustand.docs.pmnd.rs][10])
* **Lucide React**: легкие SVG icons для React. ([Lucide][11])

### Useful

* **Vercel AI SDK**: `generateText` и `streamText` для LLM text/streaming, если хочешь более стандартный AI layer. ([AI SDK][12])
* **Vercel AI Gateway**: единая точка доступа к моделям, budgets, monitoring, fallback/load balancing. ([Vercel][13])
* **OpenRouter**: unified API и fallback к разным моделям, полезно как запасной provider. ([OpenRouter][14])
* **Supabase**: база, auth, leaderboard, быстрый Next.js quickstart. ([Supabase][7])
* **Motion for React**: быстрые анимации входа, hover, transitions. ([Motion][15])
* **Kenney assets**: 2D, 3D, UI, audio, pixel assets; assets на страницах Kenney указаны как CC0/public domain, attribution not required. ([kenney.nl][16])

### Optional

* **Clerk**: если нужна красивая auth за минуты, есть Next.js quickstart и готовые компоненты. ([Clerk][17])
* **Phaser**: если тема явно arcade/platformer/top-down и есть человек, который умеет. Официальные docs и `create-phaser-game` помогут стартовать. ([docs.phaser.io][18])
* **PixiJS**: если нужен производительный 2D rendering с WebGL/WebGPU, но без полного игрового framework. ([pixijs.com][19])
* **Three.js**: только если тема требует 3D. Иначе это сияющая ловушка времени. ([threejs.org][20])

### Avoid during 5-hour hackathon

* Docker.
* Full auth + profiles + payments.
* Real-time multiplayer.
* 3D physics.
* Custom level editor.
* Complex procedural generation.
* Long agent debates.
* “Let’s rewrite this cleaner”.

# 4. Как отличиться, если все делают карточные AI-игры

Большинство сделают: карточка, 3 выбора, AI текст. Это нормально, но легко забывается. Нужен один визуальный трюк.

## Ход 1: “AI Director Camera”

Вместо обычной карточки показывай справа маленький “режиссерский монитор”:

```txt
AI DIRECTOR
Analyzing player style...
Risk profile: Aggressive
Next event: harder
Confidence: 87%
```

Это почти ничего не стоит. Просто блок UI.
Но на видео выглядит так, будто AI реально управляет игрой.

Как реализовать быстро:

```txt
После каждого выбора обновлять:
- riskProfile
- nextDifficulty
- directorNote
- confidence
```

Даже если это rule-based fallback, визуально эффект сильный.

---

## Ход 2: “Timeline of Consequences”

Под карточкой сделать горизонтальную линию из 6-8 узлов:

```txt
Choice 1 -> Choice 2 -> Choice 3 -> Crisis -> Final
```

Каждый выбор оставляет маленький след:

```txt
+Trust
-Danger
+Score
AI changed path
```

Почему работает:

* жюри видит progression без объяснений;
* игра кажется глубже;
* видео получает визуальное движение.

Это не сложная карта, а просто массив событий.

---

## Ход 3: “Ending Poster”

В конце не просто modal, а **постер концовки**:

```txt
YOUR ENDING
THE SIGNAL KEEPER

Score: 1280
Style: Risk-Taker
AI Verdict: You chose danger when safety was cheap.

[Copy Result]
[Play Again]
```

Добавь большой title, badge, градиент, одну иконку.
Финальный кадр видео сразу становится скриншотом для README.

Самая выгодная штука: концовка делает игру “больше”, чем она есть.
